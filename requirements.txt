# IT Learning Center - Dependency Requirements

This document outlines all required dependencies for the IT Learning Center Appointment System.

## Node.js Dependencies

### Server Dependencies (server/package.json)

#### Core Framework
- express@^4.18.2 - Web application framework for Node.js
- mongoose@^8.0.0 - MongoDB object modeling tool

#### Authentication & Security
- bcryptjs@^2.4.3 - Password hashing library
- jsonwebtoken@^9.0.2 - JSON Web Token implementation
- express-validator@^7.0.1 - Request validation middleware
- express-rate-limit@^7.1.5 - Rate limiting middleware
- cors@^2.8.5 - Cross-origin resource sharing

#### Environment & Configuration
- dotenv@^16.3.1 - Environment variable management

#### Email Service
- nodemailer@^6.9.7 - Email sending library

#### Development Dependencies
- nodemon@^3.0.2 - Auto-restart server on file changes

### Client Dependencies (client/package.json)

**Note**: These will vary based on your chosen frontend framework.

- @angular/common@^17.0.0

- @angular/forms@^17.0.0
- @angular/platform-browser@^17.0.0
- typescript@~5.2.2
- rxjs@~7.8.0

#### If using React:
- react@^18.2.0
- react-dom@^18.2.0
- react-router-dom@^6.20.0
- axios@^1.6.0

#### If using Vue:
- vue@^3.3.0

## Installation Commands

### Install all server dependencies:
```bash
cd server
npm install
```

### Install all client dependencies:
```bash
cd client
npm install
```

### Install global development tools (optional):
```bash
npm install -g nodemon
npm install -g @angular/cli  # If using Angular
```

## Security Notes

 **Never commit node_modules to GitHub!**
- It's already in .gitignore
- Dependencies are installed via `npm install`

 **Keep dependencies updated:**
```bash
npm outdated  # Check for updates
npm update    # Update to latest compatible versions
```

 **Audit for vulnerabilities:**
```bash
npm audit
npm audit fix
```

## Troubleshooting

### Common Issues

**Problem**: `npm install` fails with permission errors
**Solution**: 
```bash
# On Mac/Linux:
sudo npm install -g npm

# On Windows:
# Run terminal as Administrator
```

**Problem**: MongoDB connection fails
**Solution**: 
- Ensure MongoDB is running: `mongod`
- Check connection string in `.env` file
- Verify network access if using MongoDB Atlas

**Problem**: Port already in use
**Solution**: 
```bash
# Find process using port 3000
lsof -i :3000  # Mac/Linux
netstat -ano | findstr :3000  # Windows

# Kill the process or change PORT in .env
```
