# IT Learning Center - Dependency Requirements

This document outlines all required dependencies for the IT Learning Center Appointment System.

## Node.js Dependencies

### Server Dependencies (server/package.json)

#### Core Framework
- express@^4.18.2 - Web application framework for Node.js
- mongoose@^8.0.0 - MongoDB object modeling tool

#### Authentication & Security
- bcryptjs@^2.4.3 - Password hashing library
- jsonwebtoken@^9.0.2 - JSON Web Token implementation
- express-validator@^7.0.1 - Request validation middleware
- express-rate-limit@^7.1.5 - Rate limiting middleware
- cors@^2.8.5 - Cross-origin resource sharing

#### Environment & Configuration
- dotenv@^16.3.1 - Environment variable management

#### Email Service
- nodemailer@^6.9.7 - Email sending library

#### Development Dependencies
- nodemon@^3.0.2 - Auto-restart server on file changes

### Client Dependencies (client/package.json)

**Note**: These will vary based on your chosen frontend framework.

#### If using Angular:
- @angular/core@^17.0.0
- @angular/common@^17.0.0
- @angular/router@^17.0.0
- @angular/forms@^17.0.0
- @angular/platform-browser@^17.0.0
- typescript@~5.2.2
- rxjs@~7.8.0

#### If using React:
- react@^18.2.0
- react-dom@^18.2.0
- react-router-dom@^6.20.0
- axios@^1.6.0

#### If using Vue:
- vue@^3.3.0
- vue-router@^4.2.0
- axios@^1.6.0

## Installation Commands

### Quick Install (Recommended)

#### Install All Server Dependencies at Once:
```bash
cd server
npm install
```
This single command installs all dependencies listed in `server/package.json` including:
- express
- mongoose
- bcryptjs
- jsonwebtoken
- express-validator
- express-rate-limit
- cors
- dotenv
- nodemailer
- nodemon (dev dependency)

#### Install All Client Dependencies:
```bash
cd client
npm install
```

### Manual Installation (Individual Packages)

If you need to install packages individually or add them one by one:

#### Core Backend Dependencies:
```bash
cd server

# Core framework
npm install express@^4.18.2
npm install mongoose@^8.0.0

# Authentication & Security
npm install bcryptjs@^2.4.3
npm install jsonwebtoken@^9.0.2
npm install express-validator@^7.0.1
npm install express-rate-limit@^7.1.5
npm install cors@^2.8.5

# Configuration
npm install dotenv@^16.3.1

# Email Service
npm install nodemailer@^6.9.7

# Development tools
npm install --save-dev nodemon@^3.0.2
```

#### Installing bcrypt (Alternative to bcryptjs):
```bash
# If you prefer native bcrypt instead of bcryptjs:
npm install bcrypt@^5.1.1

# Note: bcrypt requires build tools on Windows
# Windows users may need: npm install --global windows-build-tools
```

#### Frontend Dependencies (Choose ONE framework):

**For Angular:**
```bash
cd client
npm install -g @angular/cli
ng new . --skip-install
npm install
```

**For React:**
```bash
cd client
npx create-react-app .
# Or manually:
npm install react@^18.2.0 react-dom@^18.2.0
npm install react-router-dom@^6.20.0
npm install axios@^1.6.0
```

**For Vue:**
```bash
cd client
npm install vue@^3.3.0
npm install vue-router@^4.2.0
npm install axios@^1.6.0
```

## Security Notes

 **Never commit node_modules to GitHub!**
- It's already in .gitignore
- Dependencies are installed via `npm install`

 **Keep dependencies updated:**
```bash
npm outdated  # Check for updates
npm update    # Update to latest compatible versions
```

 **Audit for vulnerabilities:**
```bash
npm audit
npm audit fix
```

## Troubleshooting

### Common Issues

**Problem**: `npm install` fails with permission errors
**Solution**: 
```bash
# On Mac/Linux:
sudo npm install -g npm

# On Windows:
# Run terminal as Administrator
```

**Problem**: MongoDB connection fails
**Solution**: 
- Ensure MongoDB is running: `mongod`
- Check connection string in `.env` file
- Verify network access if using MongoDB Atlas

**Problem**: Port already in use
**Solution**: 
```bash
# Find process using port 3000
lsof -i :3000  # Mac/Linux
netstat -ano | findstr :3000  # Windows

# Kill the process or change PORT in .env
```
